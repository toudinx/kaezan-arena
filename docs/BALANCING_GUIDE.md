This document defines the balancing philosophy and process for Kaezan: Arena.

The goal is not to define final numbers yet, but to establish:

target gameplay experience

metrics to observe

balancing methodology

iteration process

Balancing should be data-driven, using run telemetry generated by the game.

Core Balance Philosophy

Kaezan Arena must deliver a specific emotional curve during a run.

Runs should feel like:

calm → pressure → chaos

A good run should end with the player thinking:

“I barely survived that.”

Not:

too easy
or
unfair death
Run Duration Target

Target run duration:

180 seconds (3 minutes)

Expected outcomes:

Player Type	Expected Result
Beginner	dies around 90–120s
Average	dies around 150s
Skilled	sometimes survives full run

If everyone always survives or always dies early, balance is incorrect.

Arena Density

Arena size:

7x7 grid

Maximum comfortable density:

8–10 mobs simultaneously

More than that risks:

visual chaos
unavoidable damage
Spawn Curve

The spawn rate should follow a progressive curve.

Example target:

Time	Active Mobs
0–30s	1–2
30–60s	2–3
60–90s	3–5
90–120s	5–7
120–150s	6–8
150–180s	8–10

The player should feel that pressure increases over time.

Mob Health (TTK)

TTK = Time To Kill.

Target values:

Mob Type	Target TTK
Small	1–2 hits
Normal	3–4 hits
Elite	6–10 hits

If mobs die too fast:

combat feels empty

If mobs take too long:

arena clogs
player gets trapped
Mob Damage

Damage must create risk without inevitability.

Target damage per hit:

Enemy Type	Damage
Small	light chip
Normal	moderate
Elite	significant

The player should survive several hits but not ignore damage.

Elite Spawn Rate

Elites should appear rarely but matter.

Target:

1 elite every 40–60 seconds

Too frequent:

combat becomes unfair

Too rare:

system becomes irrelevant
Chest Frequency

Chests create build moments.

Target:

2–3 chests per run

Typical timing:

Time	Event
45s	chest
110s	chest
160s	chest

Chests should create risk/reward decisions.

Skill Power Curve

Skills must scale during the run.

Example structure:

Level	Effect
Start	basic skill
Level 2	upgrade
Level 3	upgrade
Level 4	upgrade
Level 5	upgrade

Players should feel stronger every ~30 seconds.

Card Impact

Cards should modify gameplay significantly.

Cards must:

change mechanics
not only numbers

Example strong cards:

chain explosion
life leech
spawn modifier

Cards must avoid false choices.

Movement Balance

Movement must feel responsive.

Key constraints:

player should escape small traps
movement should never feel "stuck"

Tick interval affects feel.

Possible values:

250ms (default)
150ms
100ms

Testing is required to determine the ideal value.

Tick Rate Impact

Lower tick rate increases responsiveness.

But also increases:

CPU load
network requests

Target experimentation values:

150ms
100ms
Run Telemetry

Run telemetry provides the most reliable balance signals.

Metrics collected include:

run duration
kills
elite kills
damage dealt
damage taken
min HP
cards chosen
drops

Telemetry allows balancing based on real gameplay data.

Key Balance Metrics

Important metrics to observe:

Metric	Purpose
runDuration	detect survival curve
kills	spawn pressure
minHP	difficulty spikes
damageTaken	threat level
cardsChosen	build preferences
Balance Iteration Loop

Balancing should follow this process.

Step 1:

run multiple test runs

Step 2:

collect telemetry

Step 3:

identify outliers

Step 4:

adjust numbers

Step 5:

repeat
Common Balance Problems
Arena Too Easy

Symptoms:

players always survive
no tension

Solutions:

increase spawn
increase elite frequency
increase mob HP
Arena Too Hard

Symptoms:

players die before mid game

Solutions:

reduce spawn
reduce damage
increase player survivability
Arena Too Chaotic

Symptoms:

screen unreadable
unavoidable damage

Solutions:

limit mob count
reduce projectile spam
Balance Targets for MVP

Initial MVP targets:

Metric	Target
Average run	~150s
Full run success	rare but possible
Chests per run	2–3
Max mobs	~8–10
Future Balance Layers

After core balance stabilizes:

New layers may be introduced.

Example:

difficulty tiers
ascension modifiers
elite variants
environment hazards
Final Balance Principle

Every change should answer one question:

Does this make the run more interesting?

If the change only modifies numbers without affecting gameplay decisions, it likely does not improve the game.

Recommended Next Step

Run 10–20 runs and record:

runDuration
kills
minHP
cardsChosen

Use these results to begin the first balancing iteration.
